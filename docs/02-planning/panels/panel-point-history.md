# 패널 기획서 — 포인트 히스토리

위치: 상단 네비게이션 — 상점 우측 신설 메뉴  
경로: `/wallet`  
컴포넌트 예정: `pages/PointHistory.jsx`  
구현 상태: 미구현

---

## 목적

모든 포인트 변동 내역을 공개 열람할 수 있는 히스토리 페이지.
운영 투명성 확보, 오류 추적, 경쟁 심리 강화.

## 필터 구성

| 필터 | 옵션 |
|------|------|
| 유저 | 전체 / 개별 감독 선택 |
| 거래 유형 | 전체 / 리그 보상 / 시즌 보상 / 승부 예측 / 뽑기 / 강화 / 특성 / 운영자 지급 / 초기화 |
| 기간 | 전체 / 이번 시즌 / 최근 30일 / 직접 입력 |
| 검색 | 유저명 또는 설명 키워드 |

---

## 히스토리 테이블 컬럼

| 컬럼 | 내용 |
|------|------|
| 감독 | 닉네임 (엠블럼 아이콘 포함) |
| 증감 | +NP (초록) / -NP (빨강) |
| 거래 유형 | 뱃지 형태 (예: `리그 보상`, `승부예측`, `뽑기`) |
| 설명 | 상세 내용 (예: "한국 vs 일본 승부예측 참여") |
| 시각 | YYYY-MM-DD HH:mm |
| 거래 후 잔액 | NP |

## 내 포인트 요약 카드 (로그인 시 상단 표시)

```
┌─────────────────────────────┐
│  나의 포인트                 │
│  현재 잔액:  3,310P          │
│  이번 시즌 획득: +1,820P     │
│  이번 시즌 소모: -1,200P     │
└─────────────────────────────┘
```

---

## 공개 범위

- **비로그인**: 전체 유저 히스토리 열람 가능 (읽기 전용)
- **로그인**: 상단에 내 포인트 요약 카드 추가 표시
- 운영자의 ADMIN_GRANT / ADMIN_DEDUCT 내역도 포함 (투명성)

---

## 페이지네이션

- 1페이지 = 50건
- 무한 스크롤 또는 페이지 버튼 방식 (구현 시 결정)
- 최신 거래 우선 정렬 (내림차순)

## 데이터 요구사항

| 데이터 | API |
|--------|-----|
| 전체 히스토리 | `GET /api/wallet/history?user_id=&type=&from=&to=&page=` |
| 내 포인트 요약 | `GET /api/wallet/me` |
| 유저 목록 (필터용) | `GET /api/users` |

# FC온라인 리그 사이트 기획서

## 1. 프로젝트 개요

FC온라인을 즐기는 10명 내외 친구들을 위한 자체 리그 관리 웹사이트.
경기 결과 기록, 팀 관리, 선수 검색, 통계, 명예의 전당 등을 제공한다.

---

## 2. 참여 감독

| 감독명 | 비고 |
|--------|------|
| 준현   |      |
| 종성   |      |
| 영동   |      |
| 민혁   |      |
| 삼주   |      |
| 영모   |      |
| 진수   |      |
| 기성   |      |

- 각 감독은 고유한 **앰블럼(이미지)** 을 보유
- 앰블럼은 Supabase Storage에 저장, DB에는 URL만 보관
- 우승 횟수에 따라 앰블럼 하단에 **별(★)** 이 추가됨

---

## 3. 시즌 구조

- 한 시즌당 각 감독은 나머지 7명과 **2판씩** 대결 (총 56경기/시즌)
- 경기 결과는 **참여 감독이 직접 수기 입력**
- 시즌 기간 및 활성 여부는 관리자가 설정

---

## 4. 내비게이션 메뉴

```
홈  /  리그일정  /  팀  /  선수검색  /  통계  /  명예의전당  /  선수뽑기
```

---

## 5. 페이지별 기획

### 5-1. 홈 (대시보드)

- **감독 순위표** — EPL 순위표 스타일 (승점, 승/무/패, 득실차)
- **통계 대시보드** — 감독 득점 순위, 선수 득점 순위, 득점력/수비력 지표
- **이번 시즌 정보** — 리그 기간, 활성 시즌 표시
- **우승 테마 배경** — 최근 시즌 우승 감독/팀 컨셉에 맞는 배너/배경 적용
  - 예: 해리포터의 우수 기숙사처럼 전체 분위기가 우승자에 맞게 변경

---

### 5-2. 리그일정

- 기획 미정 (추후 확정)
- 경기 일정 표시 및 결과 입력 기능
- 각 감독이 본인 경기 결과를 직접 입력

---

### 5-3. 팀

#### 팀 구성 명단

- 참여 감독 전체의 팀 스쿼드를 **그리드 레이아웃** 으로 표시
- **간략 보기**: 각 감독의 Best 11이 리스트 형태로 표시
- **상세 보기 (팝업)**: 감독이 설정한 포메이션과 선수 배치, 선수 상세 정보

#### 내 팀 설정

- 포메이션 선택 및 선수 배치 구성
- 선수 검색에서 즐겨찾기한 선수 또는 내 선수단에 추가된 선수 목록에서 선택
- **시장가치 자동 합산** — 배치된 선수들의 시세 합계 표시
- **OVR 자동 계산** — Best 11 기준 강화등급 적용 스탯 평균
- **케미 설정 (수동 선택)** — 자동 계산 없이 감독이 직접 선택
  - 강화케미효과 선택 (같은 강화등급 묶음 보너스)
  - 팀케미효과 선택 (국적/리그/클럽 연결 보너스)
- 우측 패널: 적용된 OVR, 케미 설정값 표시

---

### 5-4. 선수 검색

- 피온북 앱 / 피파인벤 웹사이트와 유사한 검색 및 필터 기능
- 포지션, 국적, 리그, 클럽, 강화등급, OVR 범위 등 필터 제공
- 추후 Nexon Open API 및 자체 DB 연동
- 기능:
  - 선수 즐겨찾기
  - 내 선수단에 추가

---

### 5-5. 통계

- 리그 전체 기준 통계 제공
  - 팀별: 득점, 실점, 승점
  - 선수별: 득점 순위
- 경기 종료 후 감독이 수기 입력한 데이터 기반

---

### 5-6. 명예의 전당

- 시즌별 리그 우승 / 컵 우승 기록
- 최대한 화려한 UI 구성 목표
- 관리자가 AI 생성 이미지를 Supabase Storage에 업로드 → 자동 반영

---

### 5-7. 선수 뽑기

- **우승 포인트** 소모형 가챠 시스템
- 포인트 적립 기준: 추후 확정 (경기 승리, 시즌 우승 등)
- 뽑기 조건: 포지션 등 필터 설정 가능
- 뽑기 애니메이션: 룰렛/슬롯 스타일로 선수 카드가 순환 후 결과 공개
- 등급 체계:
  - S급 (금색 글로우)
  - A급 (보라색 글로우)
  - B급 (라임그린 글로우)
- 결과 처리: 알림 표시 (추후 내 선수단 자동 추가 연동)

## 6. 기술 스택 요약

| 구분       | 기술 |
|------------|------|
| 프론트엔드 | React 18 + Vite + Tailwind CSS v3 |
| 백엔드     | Node.js + Express.js |
| 데이터베이스 | PostgreSQL (Supabase) |
| 인증       | bcryptjs + JWT (HttpOnly 쿠키) |
| 외부 API   | Nexon Open API (FC온라인 선수 데이터) |
| 배포       | Vercel (프론트) + Render (백엔드) + Supabase (DB/Storage) |
| 스케줄러   | node-cron (주 1회 5강/8강 시세 동기화) |

## 7. 데이터 설계 (주요 테이블)

| 테이블명              | 용도 |
|-----------------------|------|
| users                 | 감독 계정 (닉네임, 비밀번호 해시, 역할) |
| managers              | 감독 정보 (앰블럼 URL, 우승 횟수) |
| seasons               | 시즌 기간, 활성 여부 |
| matches               | 경기 일정 및 결과 |
| match_goals           | 경기별 득점 선수 기록 |
| team_squads           | 감독별 팀 구성 (포메이션, 선수, 케미 설정) |
| nexon_player_metadata | 선수 메타 정보 (이름, 포지션, 시즌 등) |
| nexon_player_prices   | 선수 강화등급별 시세 (온디맨드 캐시) |
| hall_of_fame          | 시즌별 우승 기록 및 이미지 URL |

---

## 8. 이미지 / 미디어 처리

- 모든 이미지(앰블럼, 명예의 전당 AI 이미지 등)는 **Supabase Storage** 에 저장
- DB에는 URL만 보관
- 관리자 업로드 → Storage 저장 → URL 자동 참조

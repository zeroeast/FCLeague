# FC온 리그사이트 작업 로그

## 프로젝트 개요
- **목적**: 친구 8명 FC온라인 리그 관리 웹사이트
- **스택**: React 18 + Vite 5 + Tailwind CSS v3 / Express.js / PostgreSQL(Supabase)
- **배포**: Vercel(프론트) + Render(백엔드) + Supabase(DB)
- **컨벤션**: Fluxion LMS v1.21 — kebab-case 파일, camelCase 변수, UPPER_SNAKE_CASE 상수, 함수 max 20줄, 코드 주석 영어만

---

## 현재 단계 (2026-06-17 기준)

| 구분 | 상태 |
|------|------|
| 프론트 샘플 UI | **대부분 완료** (Players·내 팀 설정·인증 연동 제외) |
| 백엔드 코드 | **골격만** (auth·players 일부, schema.sql, render.yaml) |
| DB / 배포 연동 | **미시작** (Supabase·Render 실제 연결 전) |
| 실제 데이터 | **전부 샘플** (하드코딩 → API 교체 대기) |

**한 줄 요약**: 화면은 거의 다 보이고, 이제 **Supabase → 로컬 백엔드 연결 → 로그인 동작** 순으로 백엔드를 붙이는 단계.

## 다음에 할 일 (추천 순서)

```
① Supabase 프로젝트 만들기
② schema.sql 실행
③ backend/.env 설정
④ 로컬: backend npm start → 회원가입/로그인 테스트
⑤ Render 배포 + Vercel API URL 연결
⑥ 경기 입력 API부터 프론트 연동
```

| 순서 | 작업 | 예상 시간 | 완료 기준 |
|------|------|-----------|-----------|
| **1** | Supabase 생성 + schema 실행 | 30분 | Table Editor에 users 등 테이블 보임 |
| **2** | `backend/.env` + 로컬 서버 | 20분 | `http://localhost:4000/api/health` → ok |
| **3** | 회원가입/로그인 로컬 테스트 | 30분 | Postman 또는 Login 페이지에서 가입·로그인 성공 |
| **4** | Render 배포 | 30분 | 배포 URL `/api/health` 응답 |
| **5** | Vercel 환경변수 연결 | 10분 | 배포된 프론트에서 로그인 동작 |
| **6** | 경기 입력 API + League 연동 | 2~3시간 | 경기 결과 DB 저장·순위 반영 |

> **지금 당장 1번(Supabase)** 부터 하면 됩니다. DB 없이는 auth·API 전부 막혀 있습니다.
